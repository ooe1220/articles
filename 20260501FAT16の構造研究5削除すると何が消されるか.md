# 趣旨

ファイルの削除は、実際にデータを上書きしている訳ではないことを確かめ、ファイル復元ソフトの原理を理解する。


# 検証用FAT16ファイル及びディレクトリの生成

```bash
dd if=/dev/zero of=fat16.img bs=1M count=10
sudo mkfs.vfat -F 16 fat16.img
sudo mount -o loop,uid=$(id -u),gid=$(id -g) fat16.img /mnt


```bash
# ファイル及びディレクトリの生成
echo "TEST FILE1" | sudo tee /mnt/test1.txt

printf 'HELLO%.0s' {1..512} > test_large.txt
sudo cp test_large.txt /mnt/

sudo mkdir /mnt/dir1

ls -la /mnt/


# 削除
sudo rm /mnt/test1.txt
sudo rm /mnt/test_large.txt
sudo rmdir /mnt/dir1

ls -la /mnt/
```

削除後にファイル及びディレクトリが一つも無いことを確認する

```
test@test-fujitsu:~/kaihatsu$ ls -la /mnt/
总计 20
drwxr-xr-x  2 test test 16384  1月  1  1970 .
drwxr-xr-x 20 root root  4096 11月  1 00:57 ..
```

# 考察

FAT16がファイルを削除する時、ディレクトリ要素の先頭バイトに`E5`を付けてFAT表を初期化するだけでデータそのものは初期化されない事を確認出来た。
FAT表が初期化される為、ファイルが断片化していた場合は復元が困難になるが、連続している場合はディレクトリに残る開始クラスタから順番に追って行けば復元できることが分かる。
