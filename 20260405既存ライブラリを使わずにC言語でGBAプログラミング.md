# 初めに

過去にアセンブリ言語で任天堂GBAのVRAMを単一色で塗りつぶす実験をしました。

[アセンブリでGBAプログラミング：ARM7でVRAM塗り潰し](https://qiita.com/earthen94/items/a12e05f525ac0bde6fe2)

今回は同じ処理をC言語で書き直します。


# コード及びコンパイル手順

```bash
arm-none-eabi-as header.s -o header.o
arm-none-eabi-gcc -c main.c -o main.o -nostdlib -ffreestanding
arm-none-eabi-ld header.o main.o -T link.ld -o out.elf
arm-none-eabi-objcopy -O binary out.elf out.gba
```

```link.ld
ENTRY(_start)

SECTIONS
{
    . = 0x00000000;

    .text :
    {
        *(.text)
        *(.rodata)
    }

    .data :
    {
        *(.data)
    }

    .bss :
    {
        *(.bss)
        *(COMMON)
    }
}
```


```main.c
typedef unsigned short u16;
typedef unsigned int   u32;

#define REG_DISPCNT (*(volatile u16*)0x04000000) // 画面制御レジスタ
#define VRAM        ((volatile u16*)0x06000000)

void main(void)
{
    // Mode3 + BG2
    REG_DISPCNT = 0x0403;

    u16 color = 0x07C0; // 緑
    for (int i = 0; i < 240 * 160; i++) {
        VRAM[i] = color;
    }

    while (1);
}
```
